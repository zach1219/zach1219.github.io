# 二、设计思路

## 2.1 核心理念：目标驱动，习惯支撑

系统的底层逻辑是「目标 → 任务 → 习惯 → 打卡」四层递进结构：

- 目标层（目标项目看板）：定义「我要去哪里」，用 OKR 思维设定目标和关键结果
- 任务层（子任务表）：回答「我要做什么」，把目标拆解为可执行的具体任务
- 习惯层（习惯追踪表）：解决「我每天做什么」，建立支撑目标达成的日常习惯
- 执行层（打卡日志表）：记录「我今天做了吗」，每日打卡形成闭环
知识灵感库和记账表作为辅助系统，分别解决「认知积累」和「财务透明」两个维度的需求。

## 2.2 数据关联设计

所有表都通过「关联字段」互相连接，核心关联链路如下：

- 目标项目看板 ←→ 子任务表（1个目标拆解为N个任务）
- 目标项目看板 ←→ 习惯追踪表（1个目标关联N个习惯）
- 目标项目看板 ←→ 知识灵感库（1个目标关联N条知识）
- 习惯追踪表 ←→ 打卡日志表（1个习惯对应N条打卡记录）
- 习惯/目标 ←→ 记账表（消费可关联具体习惯或目标）
这种设计的好处是：你可以在目标视图下看到所有关联的任务、习惯和知识；也可以在习惯视图下看到它支撑了哪些目标。数据之间不是孤立的，而是互相呼应。

## 2.3 设计原则

- 少即是多：每张表只保留必要字段，避免过度设计
- 闭环思维：每个目标都有任务拆解、习惯支撑、知识积累
- 数据驱动：用数字（进度、完成率）而非感觉来衡量状态
- 灵活扩展：支持按需添加新表或新字段
# 三、各表详解

## 3.1 目标项目看板

这是整个系统的「大脑」，所有目标从这里出发。

### 字段说明

- 目标名称：目标的简短描述，如「2026年身体健康管理」
- 目标类型：年度目标 / 季度目标 / 月度目标 / 项目
- 状态：未开始 / 进行中 / 已完成 / 已放弃
- 关键结果：用可量化的指标定义成功，如「每周运动3次，体重降至75kg」
- 截止日期：目标的完成期限
- 优先级：P0（最高）/ P1 / P2
- 进度：0-100% 的数字，手动或自动更新
- 拆解任务：关联子任务表中的具体任务
- 关联习惯：关联习惯追踪表中的日常习惯
- 复盘笔记：关联知识灵感库中的复盘记录
### 使用建议

- 每个季度初设定当季目标，年初设定年度目标
- 关键结果必须可量化（有数字、有截止日期）
- 每周回顾一次进度，及时调整
- 用 P0/P1/P2 区分优先级，避免所有事情都「紧急」
## 3.2 子任务表

目标的「执行单元」，把大目标拆成一个个可交付的小任务。

### 字段说明

- 任务名称：具体的任务描述
- 完成状态：已完成 / 未完成
- 截止日期：任务的 deadline
- 备注：补充说明
- 关联目标：这个任务属于哪个目标
### 使用建议

- 任务粒度建议控制在 1-3 天可完成
- 每个目标至少拆解 3-5 个关键任务
- 及时更新完成状态，保持数据准确性
## 3.3 习惯追踪表

定义你要养成的日常习惯，是目标达成的「底层操作系统」。

### 字段说明

- 习惯名称：如「冥想10min」「阅读30分钟」
- 频率目标：每天 / 每周N次 / 工作日
- 习惯类型：健康 / 学习 / 效率 / 社交
- 开始日期：习惯养成的起始日
- 状态：启用 / 暂停 / 已放弃
- 关联目标：这个习惯支撑哪个目标
- 提醒时间：设置提醒（可选）
### 使用建议

- 初期不要超过 5 个习惯，先养成再扩展
- 习惯描述要具体可执行（「阅读30分钟」比「多读书」好）
- 如果某个习惯连续一周没完成，考虑调整或放弃
## 3.4 打卡日志表

每日习惯执行的「打卡机」，记录你今天做了什么。

### 字段说明

- 日期：打卡日期
- 完成：已完成 / 未完成
- 备注：今天的感受或补充说明
- 关联习惯：打卡对应哪个习惯
### 使用建议

- 建议每天固定时间打卡（如睡前）
- 备注可以记录感受，帮助后续复盘
- 连续打卡记录是习惯养成的重要证据
## 3.5 知识灵感库

你的「第二大脑」，存储读书笔记、课程要点、灵感碎片。

### 字段说明

- 标题：知识条目的简短标题
- 类型：读书笔记 / 课程笔记 / 灵感碎片 / 播客笔记 / 文章摘要
- 内容：完整记录
- 摘要：一句话提炼核心要点
- 来源：原文链接或出处
- 重要程度：核心 / 参考 / 存档
- 复习状态：未读 / 已读 / 已内化
- 标签：多选标签，如「思维模型」「自我提升」「效率」等
- 关联目标：这条知识支撑哪个目标
- 下次复习：设定复习提醒日期
- 阅读进度：0-100%
### 使用建议

- 摘要字段非常重要，强迫自己用一句话总结，是深度理解的标志
- 定期复习「未读」和「已读」的知识，推动向「已内化」转化
- 标签系统帮助你跨主题检索，建议统一标签体系
## 3.6 记账表

个人财务的「透明窗口」，记录每笔收支并与目标关联。

### 字段说明

- 记录：消费/收入的简短描述
- 金额：具体数字（收入为正，支出为负）
- 类别：餐饮 / 交通 / 购物 / 娱乐 / 学习 / 健康 / 居住 / 社交 / 固定支出 / 其他
- 日期：交易日期
- 收支：收入 / 支出
- 支付方式：微信 / 支付宝 / 现金 / 银行卡 / 信用卡
- 关联习惯：这笔消费关联哪个习惯（如健身卡→健康习惯）
- 关联目标：这笔消费关联哪个目标
### 使用建议

- 养成即时记录的习惯，不要积压到月底
- 定期回顾支出结构，识别不必要的开支
- 关联目标后可以分析「为目标投入了多少资源」
# 四、使用方法

## 4.1 快速上手

- 第一步：打开「目标项目看板」，设定你的第一个目标（建议从年度目标开始）
- 第二步：为这个目标设定 2-3 个关键结果（可量化的指标）
- 第三步：在「子任务表」中拆解 3-5 个关键任务
- 第四步：在「习惯追踪表」中建立 2-3 个支撑目标的日常习惯
- 第五步：每天在「打卡日志表」中记录习惯完成情况
- 第六步：学习新知识时，随手记录到「知识灵感库」
- 第七步：消费时在「记账表」中记录
## 4.2 日常使用流程

### 每日流程

- 早晨：查看今天需要完成的任务和习惯
- 白天：执行任务，有灵感随手记录
- 晚上：打卡记录今天完成的习惯，记录消费
### 每周流程

- 周日晚上：回顾本周习惯完成率
- 检查未完成的任务，调整下周计划
- 回顾知识库，复习「未读」内容
### 每月流程

- 月初：设定月度目标
- 月末：复盘本月目标完成情况
- 更新目标进度
- 分析本月支出结构
## 4.3 进阶用法

### 多视图利用

- 看板视图：按状态分组，一目了然看到目标/任务的进展
- 日历视图：按日期查看任务和习惯的时间分布
- 甘特图视图：查看目标和任务的时间线
- 表单视图：创建表单，方便快捷录入数据
### 筛选与分组

- 按优先级筛选：只看 P0 任务
- 按状态分组：区分「进行中」和「已完成」
- 按类别筛选：只看「健康」类消费
### 数据联动

- 在目标视图下，可以看到所有关联的任务、习惯和知识
- 在习惯视图下，可以看到它支撑了哪些目标
- 通过关联字段，实现数据的双向导航
# 五、最佳实践

## 5.1 目标设定

- 使用 SMART 原则：具体、可衡量、可达成、相关、有时限
- 每个目标至少有 2-3 个关键结果
- 避免同时进行太多目标，建议不超过 3 个
## 5.2 习惯养成

- 从微小习惯开始（如「每天读 2 页」而不是「每天读 1 小时」）
- 使用习惯叠加法：在已有习惯后附加新习惯
- 允许偶尔的失败，但不要连续两天中断
## 5.3 知识管理

- 输入必有输出：看完一篇文章，写一句摘要
- 定期复习，推动知识从「已读」到「已内化」
- 使用标签系统，建立个人知识图谱
## 5.4 财务管理

- 即时记录，不要积压
- 定期回顾支出结构
- 关联目标，分析资源投入
# 六、常见问题

### Q: 目标太多怎么办？

A: 使用优先级（P0/P1/P2）区分，P0 不超过 2 个。如果所有事情都紧急，说明没有做好优先级排序。

### Q: 习惯总是坚持不下来？

A: 检查习惯是否太大（缩小到「不可能失败」的程度），或者是否与目标真正相关（缺乏动力）。

### Q: 知识库记了但不看？

A: 设定「下次复习」日期，利用系统的提醒功能。同时简化摘要，让复习成本降到最低。

### Q: 记账太麻烦？

A: 先从大额消费开始记录，小额可以每周集中补录。关键是养成意识，不必追求 100% 覆盖。
